# AnimateAI

AnimateAI is an AI-powered platform that generates cinematic video animations based on simple text prompts. The application uses a multi-agent orchestrated pipeline to handle video creation seamlessly.

## Architecture & Video Pipeline

The video generation pipeline uses a 3-step process combining Large Language Models (LLMs), AI Image Generation, and programmatic video rendering to turn a simple user prompt into an MP4 video.

### Tech Stack
1. **Groq (llama3-8b-8192)**: Text-to-Text inference for scene generation.
2. **Pollinations.ai**: Free, keyless API for Text-to-Image generation.
3. **FFmpeg**: Video processing, Ken Burns animation effects, and concatenation.
4. **FastAPI**: Backend routing and service orchestration.
5. **Render**: Hosting and deployment environment.

### Step-by-Step Flow

#### 1. Scene Generation (Groq)
- The backend takes the user's `title` and `prompt` and queries Groq's `llama3-8b-8192` model.
- The AI is prompted to act as a professional cinematographer and write **3 highly detailed image generation prompts** (specifying lighting, camera angle, mood, and style).
- The LLM returns a strict JSON array containing the visual descriptions and short captions for 3 scenes.
- *Fallback*: If `GROQ_API_KEY` is not set or the API fails, the system safely falls back to hardcoded cinematic scene prompts based on the project title.

#### 2. Image Generation (Pollinations.ai)
- The backend takes the 3 visual descriptions and asynchronously fetches images.
- It uses the free `image.pollinations.ai` endpoint.
- Requests are made in parallel for maximum speed.
- The generated images are temporarily saved to disk.

#### 3. Video Animation (FFmpeg)
- **Ken Burns Effect**: We apply the `zoompan` filter to slowly zoom in or out of the static images, giving them cinematic dynamic motion.
- **Text Overlay**: We add the scene's `caption` to the bottom of the video using the `drawtext` filter with dark transparent backgrounds and drop shadows.
- **Transitions**: We add `fade` in and out effects for smooth transitions between clips.
- Each image is rendered into a 4-second MP4 clip.

#### 4. Title Card & Concatenation
- We generate an animated title card using FFmpeg featuring a hue-shifting background, an oscillating title animation, and a "Generated by AI Pipeline" watermark.
- Finally, FFmpeg's `concat` demuxer stitches the 3 scene clips and the title card together into a single, seamless MP4 file.
- The file is returned to the user via a blob download to avoid cross-origin (CORS) download restrictions.

## Local Setup

### Requirements
- Python 3.11+
- FFmpeg installed and accessible in the system path
- Node.js (for frontend tooling, optional as files are static HTML/JS)

### Environment Variables
Configure the following in your `.env` file or environment:
```
GROQ_API_KEY=your_groq_api_key  # Optional, but recommended for dynamic AI scenes
```

## Running Locally
Since the backend uses FastAPI and also serves the static frontend files, simply run the application server:

```bash
cd backend
uvicorn app.core.application:create_application --host 0.0.0.0 --port 8000 --factory
```
Then visit `http://localhost:8000` in your browser.
